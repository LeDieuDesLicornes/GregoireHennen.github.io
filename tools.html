Procedural 3D Texture Generator (Unity Editor / C#)

I developed a custom editor wizard tool within Unity to instantly generate volumetric 3D texture assets powered by procedural noise math.

Three-dimensional volume preview of the generated noise inside the Inspector

Objective & Integration

The main goal of this tool was to remove the reliance on external third-party software for creating complex volumetric noise maps, providing a lightweight, in-engine solution to experiment with different resolutions and frequencies on the fly.

  • Unity Editor: Built as a custom EditorWindow utility seamlessly integrated into the editor layout.
  • Unity Mathematics: Leveraged the mathematical library to generate optimized 3D Simplex noise via the noise.snoise() function.

Editor Window Configuration

The wizard is accessible from the top menu via Tools → Générer Texture 3D Noise. The user configures the generation through three primary parameters:

Parameter Description
Size The cubic resolution of the texture across all three axes (X, Y, Z). e.g., 32³, 64³, 128³.
Frequency Controls the scale and tiling frequency of the noise pattern in 3D space.
File Name The target name for the final `.asset` file saved within the project directory.

1. Initializing the Texture3D Container

The script instantiates an empty Texture3D container using the user-defined size. It uses the RGBA32 color format. To ensure seamless looping and clean lookups, the filtering is set to Bilinear and the WrapMode is forced to Repeat.

Texture3D texture = new Texture3D(size, size, size, TextureFormat.RGBA32, false);
texture.wrapMode = TextureWrapMode.Repeat;
texture.filterMode = FilterMode.Bilinear;

2. Mathematical Simplex Sampling

Using three nested loops (X, Y, Z), the script calculates the normalized local position of every single voxel. Since snoise() outputs values in a -1 to 1 range, a standard mathematical remapping is applied to bring it into a [0, 1] range fit for color grayscale maps.

float3 pos = new float3(x, y, z) / (float)size * frequency;
float noiseValue = (noise.snoise(pos) + 1f) * 0.5f;

colors[index] = new Color(noiseValue, noiseValue, noiseValue, 1f);

3. Data Writing & UX Selection Focus

Once the pixel color array is completely populated, the texture changes are finalized via Apply(). The tool writes the native asset into the project folder using the AssetDatabase API.

For optimal user convenience, the script sets the editor's active selection directly to the newly created asset so it pops up instantly in the Inspector window.

texture.SetPixels(colors);
texture.Apply();

AssetDatabase.CreateAsset(texture, path);
AssetDatabase.SaveAssets();

Selection.activeObject = texture;

One-click generation process with automatic asset focus and highlighted inspector selection

Technical Breakdown & Future Outlooks

This tool successfully created an immediate iteration loop for generating noise variants. However, because the computational logic currently runs entirely on the CPU, generation times can scale drastically up when computing higher resolution grids (e.g., 256³).

Project Note: As the pipeline pivoted away from using this specific resource type, further development on the tool was halted. Future steps would have involved shifting the generation mathematical load onto the GPU via Compute Shaders, alongside implementing Worley and Fractal Brownian Motion (FBM) noise algorithms.

Procedural Modular Map Generator (Unity Editor / C#)

An advanced in-editor environment generator designed to speed up blockout, prototyping, and level layout assembly directly via manual scene anchor nodes.

Fully modular map automatically assembled from custom editor components and user nodes

Objective & Architecture

The primary goal is to minimize time spent on repetitive tasks by automating the placement of basic architectural modules (walls, corners, floors, and ceilings). Designers can focus directly on the level design flow instead of manual modular snapping.

The system decouples the layout data processing from the visual representation by using two specialized classes:

  • MapGenerator: The core structural logic engine controlling path data, vector remapping, and validation arrays.
  • MapGeneratorEditor: A custom inspector layer injecting instant action buttons to control scene state without running the game.

Custom Editor Actions

The tool exposes explicit controls inside the Inspector interface to guarantee rapid iteration and real-time environment validation:

Editor Button Internal Execution Logic
Générer la map Clears memory buffers, scans active scene node lists, instances modular prefabs, and structures the scene hierarchy.
Nettoyer / Effacer Performs a surgical teardown of the generated root GameObject to leave the editor hierarchy completely clean.

Custom inspector setup managing complex structures and execution actions

Core Mechanics & Algorithms

1. Anchor Paths Configuration: Every custom wall path dictates structural constraints via manual transforms. The logic tracks if the system should process closed loops (rooms) or open tracks (corridors), computing clockwise or counter-clockwise vertex layouts automatically.

2. Wall & Corner Assembly: The algorithm measures distance vectors between node handles. It snaps matching grid segments, matches orientation vectors, and handles intersections by measuring adjacent entry/exit angles to choose and rotate corner modules accurately.

3. Dual-Method Surface Filling: For closed spaces, geometric polygon inclusion checks compute bounding limits to spawn tile arrays. For paths/corridors, a progressive wave-propagation algorithm pushes rows outward until a collision node or edge restriction terminates the grid growth.

1. System Initialization & Safety Locks

The algorithm wipes existing references before starting structural calculations. It instantiates a structured GameObject parent framework and caches spatial data in a HashSet<Vector3Int> to prevent overlapping assets during multi-room construction.

public void GenererMap()
{
    NettoyerMap();
    dallesInstanciees.Clear();
    
    conteneurMap = new GameObject("Map_Generee");
    conteneurMap.transform.parent = this.transform;
}

2. Anchor Space Delta Processing

The system loops over adjacent scene node data, extracting direction normals and offsetting standard module widths to prevent mesh z-fighting. It automatically isolates distance parameters before invoking line generation methods.

Vector3 direction = pB.position - pA.position;
Vector3 directionNormalisee = direction.normalized;

Vector3 positionDepartReelle = pA.position + (directionNormalisee * longueurCoin);
Vector3 positionArriveeReelle = pB.position - (directionNormalisee * longueurCoin);
float distancePourMurs = (positionArriveeReelle - positionDepartReelle).magnitude;

3. Wave Propagation Control

To avoid script lockups or memory overflows during iterative layout expansion, an internal loop monitor serves as a hard safety cutoff. It checks bounding collision sets and kills wave progression once max limits are reached.

bool auMoinsUneVagueActive = true;
int etape = 1;

while (auMoinsUneVagueActive && etape <= seuilSecuriteMax)
{
    auMoinsUneVagueActive = false;
    // Wave generation loop logic...
}

Real-time layout building showcasing rapid cleanup, segment drawing, and hierarchical structural organization

Technical Breakdown & Future Outlooks

The tool drastically reduces production overhead by automating thousands of component placements. However, due to complex geometric filling states, occasional micro-gaps or loose edge dalles might require final manual adjustments by the level designer.

Project Note: The achieved performance met the initial design requirements perfectly. Future iterations could integrate automated gameplay asset spawning (doors, interactive triggers, gameplay points of interest) and multi-layered grid validation for multi-story level designs.

Spot Light Volumetric Mesh & Collider Generator (Unity Editor / C#)

An advanced editor utility designed to automatically generate mathematical conical meshes and synchronized convex trigger volumes based on native Unity Spot Lights.

Real-time execution inside the Scene View showing dynamic color switching, segment density adjustments, and auto-updating convex wireframes.

Objective & Technical Workflow

This tool addresses a dual requirement within our prototype pipeline: creating a lightweight visual representation of volumetric light beams and simultaneously producing precise physical collision data.

While modern pipelines like HDRP offer native volumetric fog, this tool generates a localized procedural mesh that feeds into a Mesh Collider configured as a Trigger. This allows the system to run performance-friendly gameplay detection scripts—such as triggering advanced monster AI reactions when the player sweeps them with a flashlight or structural searchlight.

  • ExecuteInEditMode: Provides instantaneous feedback directly inside the Unity Scene View upon changing light components, stripping away any need to boot the game.
  • Shader Graph Bridge: Dynamically safe-checks high-range Lux values from HDRP lights using an intensity divider to maintain flawless material transparency blending.

Configurable Script Fields

The script extends standard component data through clean, designer-friendly inspector handles:

Field Property Internal Technical Behavior
Maximum Opacity Clamped via a [Range(0f, 1f)] attribute. Controls vertex color alpha at the cone's origin tip before scaling.
Segments Dictates the mathematical resolution of the cone. Lowering this value optimizes runtime physics topology on the fly.
Intensity Divider Acts as a safety factor against high-intensity Lux values to protect visual alpha thresholds inside the target material.

1. Mathematical Bounds Mapping

The math module reads the native spotlight aperture metrics. By converting half of the spotAngle to radians and applying a tangent operation multiplied by the light's range, it yields the exact spatial radius for the cone base cap.

float farRadius = Mathf.Tan(_light.spotAngle * 0.5f * Mathf.Deg2Rad) * _light.range;
Vector3[] vertices = new Vector3[_segments + 2];
Color[] colors = new Color[_segments + 2];

2. Procedural Color & Alpha Gradients

Vertex 0 anchors at the local origin with a soft alpha. The engine then sweeps an iterative loop using Sine and Cosine coordinates to stitch outer boundary vertices, forcing their vertex alpha to 0 to generate a smooth volumetric dissipation.

vertices[0] = Vector3.zero;
colors[0] = new Color(r, g, b, _maximumOpacity / 10f);

float angle = i * (2 * Mathf.PI / _segments);
vertices[i + 1] = new Vector3(Mathf.Sin(angle) * farRadius, Mathf.Cos(angle) * farRadius, _light.range);
colors[i + 1] = new Color(r, g, b, 0f);

3. Structural Trigger Binding

Indices are systematically ordered to build side faces out of triangular strips. Once baked, the structural data updates the MeshFilter and updates the shared mesh data on the MeshCollider, forcing convex constraints for trigger operations.

triangles[j * 3] = 0;
triangles[j * 3 + 1] = j + 1;
triangles[j * 3 + 2] = (j + 2 > _segments) ? 1 : j + 2;

_collider.convex = true;
_collider.isTrigger = true;
_collider.sharedMesh = _mesh;

Technical Assessment & Optimizations

The implementation completely fulfills prototyping needs by bridging lighting components directly into physics simulation spaces. This completely eradicates manual collider layout processes for design teams.

Project Architecture Note: Because geometry rebuilding triggers during the Update sequence inside the editor, scale optimizations would incorporate a property dirty-flag check pattern. This ensures structural vertex arrays only rebuild when parameters change, rather than casting every single frame.
