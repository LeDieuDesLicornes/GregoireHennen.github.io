Générateur de Texture 3D Procédurale (Unity Editor / C#)

Développement d'un outil personnalisé sous forme d'assistant éditeur permettant de générer instantanément des assets de textures 3D volumétriques basés sur du bruit procédural.

Visualisation tridimensionnelle du volume de bruit généré dans l'Inspecteur

Objectif & Intégration

L'objectif principal de cet outil est d'éviter la création ou l'importation manuelle de textures volumétriques complexes depuis des logiciels tiers, offrant une solution rapide pour tester différentes résolutions et fréquences directement dans Unity.

  • Unity Editor : Fenêtre personnalisée (`EditorWindow`) intégrée nativement dans l'interface de développement.
  • Unity Mathematics : Exploitation de la bibliothèque mathématique pour générer un bruit Simplex 3D optimisé via noise.snoise().

Configuration de la Fenêtre Éditeur

L'outil est accessible depuis le menu supérieur de Unity : Tools → Générer Texture 3D Noise. L'utilisateur fait ses réglages via trois paramètres principaux :

Paramètre Description
Size Résolution cubique de la texture sur les trois axes (X, Y, Z). ex: 32³, 64³, 128³.
Frequency Ajuste l'échelle et la répétition du motif du bruit dans l'espace 3D.
File Name Nom donné à l'asset final qui sera sauvegardé dans le projet.

1. Initialisation du conteneur Texture3D

Le script instancie un conteneur Texture3D vide basé sur la taille demandée. Le format choisi est du RGBA32. Pour garantir des transitions fluides et répétables, le mode de filtrage est forcé en Bilinéaire et le WrapMode est configuré sur Repeat.

Texture3D texture = new Texture3D(size, size, size, TextureFormat.RGBA32, false);
texture.wrapMode = TextureWrapMode.Repeat;
texture.filterMode = FilterMode.Bilinear;

2. Échantillonnage Simplex Mathématique

À travers trois boucles imbriquées (X, Y, Z), le script calcule la position normalisée de chaque voxel. La fonction snoise() renvoie des données comprises entre -1 et 1.

Une opération de remappage mathématique standard convertit ces données dans l'intervalle [0 ; 1] afin de les appliquer proprement sous forme de niveaux de gris.

float3 pos = new float3(x, y, z) / (float)size * frequency;
float noiseValue = (noise.snoise(pos) + 1f) * 0.5f;

colors[index] = new Color(noiseValue, noiseValue, noiseValue, 1f);

3. Écriture des données et focus UX

Une fois le tableau de pixels traité, la texture est finalisée avec Apply(). L'outil génère l'asset physique dans le dossier du projet via l'AssetDatabase.

Pour maximiser le confort d'utilisation, l'outil force la sélection active de l'asset créé afin qu'il s'affiche instantanément dans l'inspecteur.

texture.SetPixels(colors);
texture.Apply();

AssetDatabase.CreateAsset(texture, path);
AssetDatabase.SaveAssets();

Selection.activeObject = texture;

Processus de génération en un clic et focus immédiat sur la ressource

Analyse Technique & Évolution

Cet outil offre un workflow idéal pour créer des variantes en quelques secondes. Néanmoins, l'exécution s'effectuant sur le CPU, le temps d'attente s'élève rapidement lors de résolutions supérieures (ex: 256³).

Note d'évolution : Le projet ayant migré vers un pipeline n'exploitant plus ce type de ressource, le développement a été suspendu. Une future itération aurait consisté à basculer la logique mathématique sur le GPU à l'aide d'un Compute Shader et d'y ajouter des bruits de type Worley ou FBM (Fractal Brownian Motion).
