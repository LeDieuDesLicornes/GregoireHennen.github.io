VFX Systems & Gameplay Feedback

This section presents the visual effects I created in Unity to support player feedback and interaction readability. Each VFX is documented through its gameplay purpose, technical setup, Unity configuration, and final in-game behavior..

Electric Breaker Feedback VFX

This electric burst is triggered every time the player raises a circuit breaker. Its goal is to make the interaction feel more responsive: the breaker action is instantly reinforced by a short electrical spark, a bright trail and a quick fade-out.

Final in-game behavior: short electric feedback when the breaker is switched back on.

VFX Breakdown

Gameplay Trigger

The effect plays when the breaker is restored, giving the player immediate visual confirmation that the interaction worked.

Particle Burst

The system uses a short burst instead of continuous emission, creating a controlled impact effect rather than a looping ambient spark.

Trail Shape

Trails are enabled to stretch the particles into lightning-like strokes, making the effect sharper and more readable.

Shader Support

A simple Shader Graph multiplies the lightning texture with vertex color, allowing the Particle System color gradient to control the final fade.

1. Particle System Timing

The effect is built as a one-shot particle burst. The main module keeps the lifetime very short, while the emission module spawns a compact group of particles at the moment of interaction.

This setup works well for a breaker because the VFX does not need to stay on-screen. It only needs to confirm the action and then disappear quickly. 


// Example gameplay hook
public void RaiseBreaker()
{
    if (_electricBreakerVFX != null)
    {
        _electricBreakerVFX.Play();
    }

    RestorePowerState();
}

Particle System setup: short lifetime, burst emission, sphere shape and yellow-to-transparent color over lifetime.

Main Particle Settings

Module Setting Purpose
Main Short lifetime and randomized start speed Creates a fast electrical pop instead of a slow smoke-like particle movement.
Emission Burst of particles at time 0 Spawns the whole spark at once when the breaker is raised.
Shape Sphere radius Distributes the spark around the breaker contact point.
Color over Lifetime Yellow to transparent Gives the effect an electrical color while naturally fading it out.
Trails Particle trails with inherited particle color Transforms small particles into visible lightning strokes.

2. Trail Renderer Setup

The Particle System renderer uses a dedicated trail material. Trails inherit the particle color, so the yellow fade defined in the Particle System also drives the opacity and timing of the lightning streaks.

This keeps the look art-directable directly from the Inspector without having to create several different materials for timing variations.

Trail and renderer configuration: particle trails, stretched texture mode and custom lightning material.

3. Lightning Trail Shader

The shader samples a black and white lightning texture, multiplies it by the particle vertex color, then sends the result to Base Color and Emission. The alpha is also multiplied by the vertex alpha, which allows the Particle System gradient to fade the texture cleanly.

This makes the shader lightweight and easy to reuse for other electric effects: the texture defines the lightning shape, while the Particle System controls color, intensity and disappearance.

Shader Graph: texture mask combined with vertex color to support particle-driven tint and alpha.

Portfolio Summary

This VFX demonstrates a small but complete gameplay feedback loop: an interaction event triggers a one-shot Particle System, the particles generate readable lightning trails, and the shader keeps the effect flexible through vertex color and alpha control.

Enemy Flash Smoke VFX

This smoke VFX is triggered when the enemy is successfully flashed. It adds a clear gameplay confirmation on top of the animation reaction: the monster briefly evaporates from strong light, similar to a vampire burning under a powerful flash.

Scene view test: the VFX is manually triggered to validate the shape around the enemy mesh.

In-game result: the flash confirms that the enemy has been hit and is reacting to light.

VFX Breakdown

Gameplay Feedback

The smoke appears when the flashlight affects the monster, making the successful flash readable for the player.

Skinned Mesh Source

The particles are spawned from the enemy Skinned Mesh Renderer, so the effect follows the creature silhouette instead of using a generic sphere volume.

Mesh Binding

A VFX Property Binder keeps the graph connected to the monster transform, allowing the smoke to stay aligned with the animated enemy.

Short Dissolve Feel

Random lifetime, velocity and size give the particles a quick evaporating motion rather than a static cloud.

1. Exposed VFX Properties

The graph exposes timing, spawn rate, random speed ranges, random size ranges and lifetime controls. This makes the effect easy to tweak directly from the Inspector without opening the VFX Graph every time.

The most important reference is the enemy body renderer. By assigning the monster Skinned Mesh Renderer, the smoke is emitted from the actual creature surface, which sells the idea that the enemy is evaporating under the flash. 


// Example gameplay hook
public void OnEnemyFlashed()
{
    if (_flashSmokeVFX != null)
    {
        _flashSmokeVFX.Play();
    }

    TriggerFlashReactionAnimation();
}

Exposed VFX Graph properties: duration, Skinned Mesh Renderer, spawn rate, random velocity, random size and lifetime range.

Main VFX Graph Settings

Area Setup Purpose
Spawn Constant spawn rate with exposed duration Controls how long the smoke burst remains active after the flash.
Initialize Set Position from Skinned Mesh Emits particles directly from the enemy body silhouette.
Velocity Random uniform X/Y/Z speed ranges Breaks up the smoke movement so it feels organic and unstable.
Output Flipbook render quad with additive blend Uses animated smoke texture frames to create a soft evaporating look.
Over Lifetime Texture index, color and size over life Fades and shrinks the smoke so it disappears quickly after the feedback moment.

2. VFX Property Binder

The VFX Property Binder sends the enemy transform into the graph. This keeps the effect connected to the monster hierarchy and avoids manually updating positions in custom code.

This is especially useful because the enemy is animated: the graph can receive the correct transform reference while still using the Skinned Mesh Renderer as the particle source.

3. Graph Structure

The graph is organized as a simple loop: exposed properties drive the spawn and initialization blocks, particles are placed on the skinned mesh, then rendered as flipbook quads with color and size controlled over lifetime.

This keeps the effect efficient and readable while still giving enough variation to make the smoke feel alive when the monster is hit by the flashlight.

Full VFX Graph overview: exposed parameters, skinned mesh spawn, random velocity, flipbook output and over-lifetime controls.

Portfolio Summary

This VFX strengthens the flashlight mechanic by combining gameplay feedback with visual storytelling. The smoke confirms that the flash worked, while the skinned mesh emission makes the monster feel like it is burning or evaporating from its own body instead of simply playing a generic particle burst.

Machine Charge Popcorn VFX

This VFX is triggered when the machine reaches 100% charge. The VFXChargeTrigger script activates two Visual Effect Graphs at the same time: one fills the background with popcorn pieces, while the second one makes popcorn jump upward to create a stronger completion feedback.

Final in-game behavior: once the machine is fully charged, the popcorn VFX ramps up, stays visible for a short moment, then fades out automatically.

VFX Breakdown

Charge Trigger

The effect starts only when the machine reaches full charge, making the completion state very readable for the player.

Two Graph Setup

One graph handles popcorn appearing in the background, while the second graph adds jumping popcorn for a more dynamic celebration.

Shared Spawn Control

Both graphs expose the same spawn parameter name, so one script can smoothly increase and decrease their intensity together.

Auto Fade-Out

After the activation window, the script lowers the spawn count back to zero and disables both VFX GameObjects.

1. Script-Driven Activation

The VFXChargeTrigger script is called by the machine when it becomes fully charged. Instead of instantly enabling a fixed burst, it animates a shared spawn parameter over time.

This makes the effect feel like it is building up with the machine: the popcorn amount ramps from 0% to 100%, holds for a few seconds, then fades back down automatically. 


public void TriggerVFXActivation()
{
    if (vfx1 != null) vfx1.gameObject.SetActive(true);
    if (vfx2 != null) vfx2.gameObject.SetActive(true);

    if (activeCoroutine != null)
        StopCoroutine(activeCoroutine);

    activeCoroutine = StartCoroutine(
        AnimateSpawnCount(0f, 1f, true, activationDuration, true));
}

Script Parameters

Parameter Value / Role Purpose
vfx1MaxSpawn 200 Controls the maximum spawn intensity for the background popcorn graph.
vfx2MaxSpawn 30 Controls the maximum spawn intensity for the jumping popcorn graph.
activationDelay 3 seconds Waits before the VFX starts ramping up, giving the machine charge event a small anticipation delay.
activationDuration 8 seconds Defines how long the spawn count takes to reach full intensity.
deactivationDelay 8 seconds Keeps the VFX visible at full intensity before the automatic fade-out starts.
deactivationDuration 1 second Quickly fades the spawn count back to zero and disables the VFX objects.

2. Background Popcorn Graph

The first VFX Graph is used for the popcorn that appears in the background. It uses a box-shaped spawn area, long particle lifetime and mesh output to populate the scene with many small popcorn meshes.

The graph also uses random size and random rotation, so the duplicated popcorn pieces do not look like repeated identical meshes.

Background popcorn graph: controlled spawn count, box position shape, mesh output, random size and random rotation.

3. Jumping Popcorn Graph

The second VFX Graph uses a stronger upward velocity and gravity, making popcorn pieces jump and fall back down. This creates the more dynamic part of the feedback when the machine reaches 100% charge.

Collision is enabled in the update stage, so the particles can interact with the local box area instead of simply flying through the effect volume.

Jumping popcorn graph: randomized direction, gravity, collision box, mesh output and shared spawn control.

4. Shared Spawn Ramp

Both VFX Graphs expose a float parameter named SpawnRate. The script writes different maximum values to each graph, but drives them using the same normalized percentage.

This keeps both layers synchronized while still allowing the background popcorn and jumping popcorn to have different densities. 


float currentPercent = Mathf.Lerp(
    initialPercent,
    endPercent,
    elapsedTime / currentTransitionDuration);

if (vfx1 != null)
    vfx1.SetFloat(spawnParameterName, currentPercent * vfx1MaxSpawn);

if (vfx2 != null)
    vfx2.SetFloat(spawnParameterName, currentPercent * vfx2MaxSpawn);

Portfolio Summary

This VFX combines gameplay state, scripting and Visual Effect Graph layering. The charge script turns a 100% machine state into a readable visual reward, while the two graph setup separates ambient background filling from more energetic jumping popcorn feedback.
